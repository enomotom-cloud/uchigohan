# 02. ペルソナとユーザーストーリー

## テーマ再解釈

本ハッカソンのテーマ「**人をダメにする**」を、私たちは「**サボらせる**」として再解釈します。  
「考える」「決める」「気にする」をAIに委ね、「作る」「食べる」「分かち合う」を人間に残すことで、料理に込められる愛情を守ります。以下のペルソナとストーリーは、すべてこの再解釈の上に立っています。

---

## ペルソナ

### 3ペルソナと3つの価値軸の対応

うちごはんの3つの価値軸は、3ペルソナそれぞれに異なる重みで現れます：

| ペルソナ | 主に得る価値 | 補助的に得る価値 |
|---|---|---|
| 田中美咲（共働きの母） | **現在性**：今日の自分の疲労に合った献立 | チャレンジ：得意料理ループからの脱却 |
| 田中雅子（一人暮らしの母） | **愛情**：娘と同じ献立を食べる連動感 | 現在性：体調・薬時間との同期 |
| 田中健司（離れて暮らす弟） | **愛情**：姉に負担をかけず、母を想える | （見守りに特化、献立は美咲に委譲） |

---

### Persona 1：田中 美咲（32歳・女性・ペルソナ主軸）

| 項目 | 内容 |
|---|---|
| 家族構成 | 夫（35歳・会社員）、子（5歳・男児） |
| 居住 | 東京都内、3LDKマンション |
| 職業 | IT企業マーケティング職、リモート＋出社ハイブリッド |
| 平日リズム | 7:30保育園送り→9:30勤務開始→18:30退勤→19:30帰宅→21:00寝かしつけ |
| ストレス源 | 「今日何作る？」を毎日30分悩む。同じメニューが続いて罪悪感 |
| 価値観 | ちゃんとした母親でいたい、でも限界がある、手料理は続けたい |
| デジタル習熟度 | 高（業務でAIツール使用経験あり） |
| LINE使用 | 日常的に独り言・雑談を投稿する習慣あり |
| 親世帯との関係 | 千葉郊外の母（62歳）が一人暮らし、月1回電話 |

**心の声**：「ナッシュ頼んだら楽だろうけど、子どもには手料理を食べさせたい。でも毎日考えるのがしんどい」

#### ビフォー（サボれない世界）

夜19:30、保育園から帰ってリビングに荷物を放る。息子が「ママー、おなかすいた」と寄ってくる。LINE のトーク画面を開いて「今日疲れた…何も考えたくない」とつぶやくが、誰も拾わない。冷蔵庫を開けて立ち尽くす3分間、頭の中で独り言が止まらない。「鶏肉と豆腐とキャベツ。何作ろう。今日揚げ物食べたから魚かな。でも子どもは魚あんまり…千葉のお母さん最近野菜足りてるかな…明日出張なのに買い物も…」結局、子にレトルトカレーを温めて、自分は先に食べる。

#### アフター（サボれる世界）

帰り道、LINE で「今日きつかった〜何も考えたくない」とつぶやく。これだけ。10分後、うちごはん Bot が返信する。

> お疲れさま。今日は冷蔵庫の鶏むね・豆腐・ニラだけで作れる、楽天レシピ「塩レモン鶏炒め」を見つけたよ。15分で完成。買い物ゼロで完結します。
> お母さんも今朝『野菜食べたい』と言ってたので、千葉のキッチンにも同じ献立を提案しました。
> これでいい？　【👍 OK】 【🔄 別の案】 【🌙 全部おまかせ】

スタンプ感覚で「OK」。家に着く。何も買わなくてよかった。エプロンをつけて鍋に火をかける。息子が横で「コショウ振る！」と手伝う。夫が帰宅。「ただいま、あ、いい匂い」。3人で食卓を囲む。22:00、千葉の母から「今日、美咲ちゃんと同じごはん食べたよ」と LINE が届く。美咲は思わず鍋を見つめる。

**残るもの**：料理、食卓、家族。**奪われたもの**：考えること、それだけ。

### Persona 2：田中 雅子（62歳・女性・美咲の母）

| 項目 | 内容 |
|---|---|
| 家族構成 | 夫と死別、千葉郊外で一人暮らし |
| 健康 | 軽度の高血圧、糖尿病予備軍 |
| 食生活 | 同じ献立の繰り返し、煮物多め、夜は簡素 |
| デジタル習熟度 | 中（LINE・写真・YouTube程度、新しいアプリは苦手） |
| LINE使用 | 娘に教わって使えるが、テキスト主体 |
| 価値観 | 子どもに心配かけたくない、自立していたい |
| 心配事 | 自分が食べているものを娘に知られたくない部分もある |

**心の声**：「娘は忙しいから連絡控えてる。でも本当は私の食事のことも気にかけてほしい。監視はされたくないけど」

#### ビフォー（サボれない世界）

1人だと食事を作る気が起きない。スーパーの惣菜パックが続く。塩分高めだけど、まあ。薬は朝食後…と思っていたら昼になっている。娘の美咲は共働きで忙しい。電話したいけど、迷惑だろう。息子の健司は大阪、出張続きで連絡もない。テレビをつけて、独り言で「今日も美味しいね」とつぶやく。

#### アフター（サボれる世界）

朝、LINE に「寒いね」と独り言を入れる。それだけで、AIが推定する：「気温低下＋単調な食事継続」。返信が来る。

> 今日は温かい煮物どうですか。冷蔵庫の大根と鶏で15分。
> 美咲ちゃんも今夜、同じ献立を提案しました。お薬の時間と食事を合わせると19:30が良さそうです。

19:30、テレビをつける前にスマホに美咲の食卓写真が届く（任意配信）。「同じごはん食べてるよ」のメッセージ付き。雅子は写真を返さない。**返さなくていい設計**だから。「心配をかけない」を、サボれる。

### Persona 3：田中 健司（35歳・男性・美咲の弟）

| 項目 | 内容 |
|---|---|
| 家族構成 | 大阪在住、独身 |
| 職業 | 製造業、中堅エンジニア、出張多い |
| 母との関係 | 月1回電話、年2回帰省 |
| 心配事 | 母の食事がちゃんとしているか、姉に任せきりで申し訳ない |

**心の声**：「お母さんが今週ちゃんと食べてるかだけ知れたら、それだけで気が楽になる」

#### ビフォー（サボれない世界）

大阪在住、出張続き、独身。母のことは気がかりだが、何を話していいかわからない。電話しても「元気よ」「あんたは？」で終わる。姉の美咲が全部背負ってくれている、という申し訳なさ。「今度実家帰る」がもう半年続いている。仕事と罪悪感のサンドイッチ。

#### アフター（サボれる世界）

週次でAIサマリが LINE に届く。

> 今週の田中雅子さん：食事7日中6日完食、塩分基準内、薬時間±15分以内、姉・美咲との食卓連動5日、睡眠リズム良好。
> 詳細はプライバシー保護のため非表示。悪い知らせがあれば即時通知します。

詳細は3親等として見えない設計。だが「悪い知らせがあれば即時通知」と書いてある。安心。それで、月に1度、何もない日に電話する。**「最近どう？」が、罪悪感じゃなく、純粋な関心からの言葉になる。** 親孝行を、サボれる。

## ユーザーストーリー

### A. 美咲（主軸）視点

#### A-1. LINE雑談から体調・予定を伝えるのを、サボる
> **As a** 共働きの母  
> **I want to** LINEに「疲れた」「忙しかった」など普通の独り言を投げるだけで疲労度を伝えたい  
> **so that** 「体調入力」という作業すら意識せず日常を過ごせる

**受け入れ基準**：
- LINE発言1回からBedrock + Comprehendで疲労度1-10を推定
- 過去24時間のメッセージを集約して時系列傾向を加味
- ユーザーが「疲労度を入力する」というUIを一切持たない
- 推定根拠を求められた時だけ「今日のメッセージから推定しました」と理由を返答

---

#### A-2. 退勤前のメニュー受信
> **As a** 共働きの母  
> **I want to** 退勤直前にLINEで当日のメニューが届いてほしい  
> **so that** 帰宅後すぐ料理に取り掛かれる

**受け入れ基準**：
- 退勤打刻 or カレンダー終了10分前にLINEプッシュ
- メニュー1案＋代替案2案を提示（疲労度・冷蔵庫・天気を反映）
- 各メニューに楽天レシピへのリンクを添付
- 「これにする」を1タップで決定
- 「食材足りない」場合はTier別に対応案を提示（後述）

---

#### A-3. 冷蔵庫の食材登録（LINE写真送信のみ）
> **As a** 入力疲労を避けたいユーザー  
> **I want to** LINEで冷蔵庫の写真を1枚送るだけで食材が登録される  
> **so that** 入力負荷で続かなくならない

**受け入れ基準**：
- LINEで写真送信→S3→Bedrock Vision/Rekognitionで食材抽出
- 写真1枚で5品目以上を80%精度で認識
- 認識結果はLINEメッセージで返却、必要に応じて修正可能
- レシート画像からの追加登録もサポート

---

#### A-4. 家族の好みと予定の反映
> **As a** 子の親  
> **I want to** 子のアレルギー・好き嫌い・夫の予定を一度登録すれば自動反映されてほしい  
> **so that** 毎回考慮しなくていい

**受け入れ基準**：
- 家族プロフィールで好き嫌い・アレルギー・苦手食材を登録
- 配偶者のカレンダー連携で帰宅予定時刻・夕食有無を取得
- メニュー提案時に「子NG食材」「夫不在で簡素化OK」を自動加味

---

### B. 雅子（母）視点

#### B-1. 自分の食事を娘に「ちょうどいい深さ」で見せたい
> **As a** 一人暮らしの高齢者  
> **I want to** 娘・息子それぞれに違う深さで自分の食事を見せたい  
> **so that** 監視されず、適度に心配される

**受け入れ基準**：
- 親等別の公開設定UI（1親等：詳細／2親等：サマリー／3親等：安否のみ）
- デフォルトは「2親等まで栄養サマリーのみ公開」（プライバシー優先）
- 親世代向けは「写真撮るだけ」UI、設定は娘が代行可

---

#### B-2. 美咲との食卓連動を、心配されずに体験したい
> **As a** 一人暮らしの高齢者  
> **I want to** 娘と同じ献立を提案されて、同じものを食べる体験をしたい  
> **so that** 一人の食卓に温かさが戻る

**受け入れ基準**：
- 美咲が「OK」を返すと、雅子側にも同献立が提案される
- 雅子の冷蔵庫に必要食材があれば即提案、なければ近所スーパー案内
- 美咲が食卓写真を共有すれば（任意）、雅子のLINEに届く
- 雅子側からの返信は不要設計（一方向OK）

---

#### B-3. 月次の健康アドバイスを受けたい（Phase 2）
> **As a** 高血圧傾向の60代  
> **I want to** 月に1回、栄養の偏りと改善案を受け取りたい  
> **so that** 通院頻度を減らしたい

**受け入れ基準**：
- 月末に「先月の栄養バランス傾向」レポートが自動生成
- 「塩分やや多め」「たんぱく質不足気味」など平易な表現
- 改善案は「次の買い物で○○を加えてみては」と具体的
- 医療判断は行わず、「気になったら主治医に相談」明記

---

### C. 健司（弟）視点

#### C-1. 母の食事をゆるく確認したい
> **As a** 遠方の家族  
> **I want to** 母が今週ちゃんと食事してるかだけ知りたい  
> **so that** 心配の負担を軽くしつつ干渉せずにいたい

**受け入れ基準**：
- 母から「2親等公開」承認を受けた状態で、週次サマリーをLINE通知で受信
- 「今週はバランス良好」「やや簡素な食事が続いています」など3段階表現
- 詳細は閲覧不可（母のプライバシー保護）
- 異常時のみアラート（「3日連続で食事記録なし」など）

---

### D. 共通：親等モデルとプライバシー

#### D-1. 親等別公開範囲の明示
> **As a** プライバシーに敏感なユーザー  
> **I want to** 誰がどこまで自分の食情報を見られるか視覚的に把握したい  
> **so that** 安心して使い続けられる

**受け入れ基準**：
- 「家族マップ」画面で親等ごとの公開範囲を色分け表示
- 公開設定変更時は事前確認＋家族メンバーへ変更通知
- データ削除・退会時は関連家族データもまとめて削除

## ユーザージャーニー（美咲の典型的な平日）

```
07:00  起床 — LINEに「眠い…」と独り言（疲労度を察知される）
07:05  AIが朝の発言＋天気＋冷蔵庫から候補メニュー3案を内部生成
       ↓
       (日中：通常勤務 — LINEでの何気ない発言が疲労度に反映される)
       ↓
17:50  PCのカレンダー終了10分前 → LINEに通知
       「今日のメニュー：豚肉と白菜のミルフィーユ鍋（白菜が今日まで）」
       「楽天レシピで作り方を見る」リンク付き
17:51  「これにする」1タップ
18:30  退勤
19:30  帰宅 → スマホでレシピを見ながら調理
20:00  食卓を家族で囲む（うちごはんはここでお役御免）
20:30  食卓写真をLINEで母に送信（任意）→ 母にも届く
21:00  寝かしつけ後、明日の冷蔵庫写真を1枚撮影してLINEへ送信
22:00  千葉の母から「今日、同じごはん食べたよ」とLINE
```

**変化の核**：「考える」「調達する」を完全にAIに渡し、「作る」「食べる」「分かち合う」を人間に残す。
