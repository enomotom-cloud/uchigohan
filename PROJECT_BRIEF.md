## サービス：うちごはん

- **メインキャッチ**: 思考はAIに、味付けは愛で。
- **サブコピー**: AIが整える、あなたが愛する。
- **設計思想**: 愛情を守る、良い怠惰。
- **一行説明**: 共働き家庭の「今日何作ろう」を考えなくする献立AI。家族の冷蔵庫・仕事疲労・体調をリアルタイム統合。親等モデルで親世帯・兄弟まで自然拡張。

## コアコンセプト：良い怠惰／悪い怠惰

```
AIに渡す（=良い怠惰）
  献立を考える、選ぶ、調整、栄養計算、買い物リスト
              ↓
人間に残す（=ダメにしない）
  作る、味見、盛り付け、食卓、分かち合い
```

「**AI完全代替時代だからこそ、代替してはいけない領域の設計**」が差別化の核。ナッシュ等は料理ごと外注、うちごはんは「考える疲労だけ削る」。

## アーキテクチャ要点

10 Unitに分解（U1-U10）。各Unitに「AI担当」と「人間担当」の比率を明示。**U6 Cooking Navigator が最もAI関与度が低い（30%）= 意図的設計**。料理を作る時間こそ人間に残す愛情の源泉。

主要AWSサービス: Bedrock (Claude Sonnet 4.6) / Lambda / API Gateway / AppSync / Cognito / DynamoDB / S3 / OpenSearch / Comprehend Medical / EventBridge / SNS / Polly / Rekognition

## ビジネスモデル：B2B2C

| 売り先 | 想定単価 |
|---|---|
| 健保組合（健康経営） | 月500-1000円/従業員 |
| 大企業福利厚生 | 月300-800円/従業員 |
| 自治体（介護予防） | 補助金スキーム |
| 食品EC連携（Oisix等） | レベシェ15-20% |
| 栄養士マーケット手数料 | 取引15-20% |

価値計算：献立思考20分/日×365日=122時間/年 → 時給2,500円換算で約30万円/年の認知労働コスト削減。月額500-800円のサービスで投資対効果30倍以上。

## 主要な過去議論サマリ

1. **当初の調査範囲**: 世代別（Z/ミレニアル/X/団塊Jr/シニア）×プライベート/仕事×2010 vs 2020-26で網羅的に不満・面倒事を分析
2. **9案からの絞り込み**: 介護コンシェルジュ/家事連鎖/メール処理など → ユーザー指摘で再検討
3. **B2C面白系の検討**: ユーザーが「映画×お菓子AI」例を提示 → 過去優勝（KanpAi/Ctrl+Cat等）の勝ちパターンと一致
4. **B2B2Cへの転換**: 単純なB2Cでは事業性弱・B2Bは「人をダメにする」テーマと不整合 → B2B2C福利厚生型を採用
5. **介護＋共働き＋家族の統合**: 親等モデルで自然に拡張、Phase分けで野望を見せる構造に
6. **ネーミング**: おかんAI（性別固定でNG） → うちごはん（中立・温かい）に決定
7. **キャッチコピー**: 「思考はAIに、味付けは愛で」採用（AI/愛の同音遊び、Human-AI調和を表現）
8. **テーマ解釈**: 「ダメにしていい部分／ダメにしてはいけない部分」を意図的に分ける哲学的アプローチ
